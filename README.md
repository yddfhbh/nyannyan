# Discord 문제 업로드 봇

`1509460473060261949` 채널에서 `/업로드` 명령어로 문제를 올리면, 선택한 단원 번호에 맞는 채널로 봇이 문제를 출력합니다.

출력 예시:

```text
24-2/일반, 35번 -1
정답: 미선택
```

봇은 출력된 메시지 아래에 `1️⃣ 2️⃣ 3️⃣ 4️⃣` 반응을 붙입니다. 누른 번호를 정답으로 인식해서 봇 메시지의 `정답:` 줄을 수정합니다.

## 준비

1. Discord Developer Portal에서 Bot을 만듭니다.
2. Bot 설정에서 `MESSAGE CONTENT INTENT`를 켭니다.
3. OAuth2 URL Generator에서 `bot`, `applications.commands` scope를 선택하고 서버에 초대합니다.
4. 봇 권한을 줍니다.
   - 채널 보기
   - 메시지 보내기
   - 파일 첨부
   - 반응 추가
   - 메시지 기록 읽기
   - 메시지 관리

## 설정

`.env.example`을 참고해서 `.env`를 만듭니다.

```env
DISCORD_TOKEN=봇_토큰
DISCORD_GUILD_ID=
SOURCE_CHANNEL_IDS=1509460473060261949
PROBLEM_CATEGORY_ID=1509459977801044029
UNIT_CHANNEL_PREFIX=
UNIT_CHANNEL_MAP=
```

기본 동작은 단원 `1`이면 이름이 `1`인 채널, 단원 `2`이면 이름이 `2`인 채널을 찾습니다.

채널 이름이 숫자가 아니면 직접 매핑할 수 있습니다.

```env
UNIT_CHANNEL_MAP=1:123456789012345678,2:234567890123456789
```

`/업로드`가 바로 안 보이면 `DISCORD_GUILD_ID`에 서버 ID를 넣고 봇을 다시 실행하세요.

## 명령어

`/업로드`

- `년도`: `26-1`, `25-2`, `25-1`, `24-2`, `24-1`, `23`
- `문제번호`: `1`부터 `60`
- `단원`: `1`부터 `12`
- `파일`: 문제 이미지 또는 파일

`/이동`

- `기존단원`: 옮길 문제의 현재 단원
- `새단원`: 옮겨갈 단원
- `미리보기`: 켜면 실제로 옮기지 않고 대상 개수만 확인

`/이동`은 `1509459977801044029` 카테고리 안의 채널들을 스캔합니다. 메시지 제목의 `24-2/일반, 35번 -1, 2번` 같은 부분에서 `-1`을 단원으로 읽고, 맞는 문제를 새 단원 채널로 복사한 뒤 원본을 삭제합니다.

기존 메시지 뒤에 `, 2번`처럼 정답이 이미 붙어 있으면, 옮긴 메시지도 그 정답을 그대로 유지하고 `1️⃣ 2️⃣ 3️⃣ 4️⃣` 반응만 다시 붙입니다.

문제 하나만 빠르게 옮길 수도 있습니다. `메시지 관리` 권한이 있는 사람이 봇이 올린 문제 메시지에 답장으로 `1`부터 `10` 사이 숫자만 입력하면, 그 문제를 해당 단원 채널로 옮기고 입력한 답장 메시지는 지웁니다.

## 실행

```bash
npm install
npm start
```

VM에 올려서 계속 실행하는 방법은 [VM_DEPLOY.md](VM_DEPLOY.md)를 보면 됩니다.
